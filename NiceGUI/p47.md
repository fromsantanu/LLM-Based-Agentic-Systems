# **Lesson 47: Deploying a NiceGUI App on Cloud Platforms (DigitalOcean, AWS Lightsail, Hostinger VPS, etc.)**

In this lesson, you will learn **how to move your NiceGUI + FastAPI project from your laptop to the cloud** so it becomes a real online application.

To make everything simple, we follow the same logic as teaching someone how to install a fridge in a new house:

* First, you need the **house** → (Cloud server/VPS)
* Then connect **electricity + water** → (SSH, firewall, ports)
* Then place your **fridge** → (Upload your project)
* Then start it → (Run code with uvicorn / Docker)
* Then allow visitors → (Nginx + domain + HTTPS)

We’ll learn these steps in a friendly, beginner-friendly way.

---

# **1. What we will deploy**

A simple NiceGUI + FastAPI app (from earlier lessons):

* UI: NiceGUI
* Backend APIs: FastAPI
* Running with `uvicorn`
* Optionally running inside Docker
* Protected and exposed using Nginx

This model works the same for:

* **DigitalOcean Droplet**
* **AWS Lightsail**
* **Hostinger VPS**
* **Linode, Vultr, Hetzner**, etc.

Cloud differences are small — the steps remain identical.

---

# **2. Step-by-step deployment (works on all cloud platforms)**

We divide the whole process into **7 easy stages**.

---

## **Stage 1: Create a cloud server (VPS)**

Choose one:

### **1. DigitalOcean**

* Create a “Droplet”
* Choose **Ubuntu 22.04**
* Small size is enough: 1 GB RAM / 1 CPU
* Add SSH key (recommended)

### **2. AWS Lightsail**

* Create an “Instance”
* Choose **Linux/Ubuntu**
* Lightsail is simpler than full AWS EC2

### **3. Hostinger VPS**

* Pick a VPS plan
* Choose **Ubuntu 22.04**
* Hostinger gives a nice dashboard

**All three** give you:

* Public IP address
* SSH access
* Root or sudo privileges

---

## **Stage 2: Connect to your server (SSH)**

On your computer:

```bash
ssh root@YOUR_SERVER_IP
```

(or your user name, like `ubuntu@...` depending on cloud provider)

If you are using Windows, you can use:

* PowerShell
* Git Bash
* Windows Terminal

You are now “inside” the server.

---

## **Stage 3: Install system packages**

Run this on the server:

```bash
sudo apt update
sudo apt install python3-pip python3-venv nginx
```

You now have:

* Python 3
* pip
* Virtual environment support
* Nginx web server

---

## **Stage 4: Upload your project**

Go to your project folder on your laptop.

### Option 1 — Using `scp` (copy files over SSH)

```bash
scp -r . root@YOUR_SERVER_IP:/srv/nicegui-app
```

### Option 2 — Using GitHub (recommended)

On your server:

```bash
cd /srv
git clone https://github.com/yourname/your-repo.git nicegui-app
```

### Option 3 — Using Hostinger File Manager

Upload ZIP → Extract → Done.
(Simple for non-technical users.)

---

## **Stage 5: Run your app on the server**

Inside your cloud server:

```bash
cd /srv/nicegui-app
python3 -m venv venv
source venv/bin/activate

pip install -r requirements.txt
```

Run the app:

```bash
uvicorn main:app --host 0.0.0.0 --port 8000
```

Try visiting:

```
http://YOUR_SERVER_IP:8000
```

You should see your NiceGUI page.

---

## **Stage 6: Add Nginx (production gateway)**

Just like in Lesson 46.

Create:

```bash
sudo nano /etc/nginx/sites-available/nicegui_app
```

Add:

```nginx
server {
    listen 80;
    server_name YOUR_DOMAIN_NAME_OR_IP;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

Enable it:

```bash
sudo ln -s /etc/nginx/sites-available/nicegui_app /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

Now open your browser:

```
http://your_domain_or_ip/
```

Your NiceGUI app appears!

---

## **Stage 7: Add free HTTPS (Let’s Encrypt SSL)**

If you have a domain:

Install Certbot:

```bash
sudo apt install certbot python3-certbot-nginx
```

Run:

```bash
sudo certbot --nginx -d yourdomain.com -d www.yourdomain.com
```

Now your site becomes:

```
https://yourdomain.com
```

Secure and production-ready.

# **3. Optional: Deploy using Docker (much easier)**

Some developers prefer Docker because:

* Same environment everywhere
* No “dependency conflicts”
* Easy to restart & scale

You only need:

### **Dockerfile** (from previous lesson):

```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
EXPOSE 8000
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

### Then on the server:

```bash
docker build -t nicegui-app .
docker run -d -p 8000:8000 nicegui-app
```

Nginx stays the same as before.

# **4. Optional: Run permanently (so it survives reboot)**

### **Option 1: systemd**

Create:

```bash
sudo nano /etc/systemd/system/nicegui.service
```

Add:

```
[Unit]
Description=NiceGUI App
After=network.target

[Service]
User=root
WorkingDirectory=/srv/nicegui-app
ExecStart=/srv/nicegui-app/venv/bin/uvicorn main:app --host 0.0.0.0 --port 8000
Restart=always

[Install]
WantedBy=multi-user.target
```

Enable:

```bash
sudo systemctl enable nicegui
sudo systemctl start nicegui
```

### **Option 2: Docker + restart**

```bash
docker run -d --restart always -p 8000:8000 nicegui-app
```

# **6. Summary (simple idea)**

Deploying your NiceGUI app to the cloud is just **7 simple steps**:

1. Create a VPS (Droplet/Instance)
2. SSH into the server
3. Install Python + tools
4. Upload your project
5. Run it with uvicorn
6. Put Nginx in front
7. Add HTTPS for security

Once you do it one time, you can deploy any future project easily.
